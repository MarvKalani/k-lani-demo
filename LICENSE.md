Third-Party Architectural References & Responsible Use

This repository implements native Rust inference engines based on the following
research and open-source blueprints:

* **OmniVoice (k2-fsa) & Higgs-Audio (Boson AI):** Core architecture for
  multilingual masked-audio generation and DAC-decoding.
* **Gemma 4 (Google DeepMind):** Foundational architectural concepts for
  Vision-Language-Model (VLM) integration and reasoning capabilities.
* **Qwen (Alibaba Cloud):** Structural inspiration for the LLM backbone and
  multi-modal token processing.
* **MiniCPM-V / MiniCPM-o (OpenBMB):** Conceptual reference for compact
  vision-language and omnimodal model workflows, including image/video prompt
  formatting and OCR-style evaluation fixtures. Local interoperability testing
  also used OpenBMB MiniCPM-V 4.6 reference artifacts, exported traces, and
  validation/benchmark image fixtures solely to verify native parity and
  performance behavior under the applicable upstream terms.
* **Lucebox Hub / Luce megakernel work (Luce-Org):** Architectural prior-art
  reference for native GPU megakernel execution and compact local inference
  orchestration. This repository does not copy Lucebox Hub source code; it only
  acknowledges the implementation idea and comparative design direction.
* **Tinygrad (tiny corp):** Philosophy for minimalist tensor operations and
  efficient backend design.
* **Larql:** Conceptual approach to structured model interaction and
  knowledge-graph querying.

Ethical Considerations & Responsible AI:
In alignment with the Gemma 4 and Qwen responsible use guidelines, the
K-Lani-Core is designed with safety and transparency in mind. Users and
developers are encouraged to:
1. Implement appropriate content safety safeguards.
2. Adhere to privacy regulations regarding personal data.
3. Monitor for and mitigate socio-cultural biases in generated outputs.

All logic was independently re-implemented in Rust to create a high-performance,
dependency-light environment optimized for robotics (PiCar-X).
